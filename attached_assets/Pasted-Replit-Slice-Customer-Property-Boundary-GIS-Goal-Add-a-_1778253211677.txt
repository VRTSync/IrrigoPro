Replit Slice: Customer Property Boundary (GIS)
Goal
Add a per-customer property boundary feature to IrrigoPro:

On each customer profile, give the user a slick drag-and-drop zone for uploading a property boundary as .kml or .kmz.
Render that boundary as a glowing purple polygon (animated marching-ants stroke + soft outer glow + translucent fill) on top of an Esri World Imagery satellite + labels base layer.
Use the saved boundary as the centering mechanism for every map shown after a customer is selected — work order location picker, estimate wizard location picker, standalone billing sheet location picker, and the customer site-maps map. The polygon also renders as a translucent overlay on those maps so techs can see they're working inside the right parcel.
Visually polish all location pickers (slicker drop zones, gradient headers, satellite imagery, layer switcher).

No code changes have been made yet. This is the scoped plan.

Current state (read first)

Stack: React 18 + Vite + TS, Express + Drizzle (Postgres/Neon), TanStack Query, Tailwind, Leaflet 1.9.4, xml2js + browser DOMParser.

Decisions already made (don't re-ask the user)

File formats accepted: .kml and .kmz only.
Polygon style: glowing purple (#B026FF) — stroke + outer glow halo + translucent fill + animated marching-ants dash.
Base layer: Esri World Imagery + Esri Reference Labels overlay (free, keyless, ToS-clean) with a layer switcher to Streets / Light. This replaces the current Google Satellite tiles in LocationPicker and is added as a new option in ColorCodedMapViewer.
Boundary scope: every customer-scoped map auto-centers on the saved boundary and renders the overlay.

2. Reusable client libraries
Create three new files in client/src/lib/:
leaflet-base-layers.ts — factories for createSatelliteLayer() (Esri World Imagery), createReferenceLabelsLayer() (Esri reference labels), createStreetsLayer() (OSM), createCartoLightLayer(), plus addSatelliteHybrid(map, { withLabels }) and createBaseLayers() (record for L.control.layers).

boundary-style.ts — exports:

Color constants: BOUNDARY_PURPLE = "#B026FF", plus glow/fill rgba variants, and a yellow palette for future use.
ensureBoundaryStyles() — injects a single <style> tag (id irrigopro-boundary-styles) once per session containing .boundary-glow-outer (drop-shadow filter), .boundary-glow-inner (animated marching ants), and @keyframes boundary-march { to { stroke-dashoffset: -24; } }.
drawPropertyBoundary(map, geojson, { color: "purple" | "yellow", animated, filled, interactive }) — adds three layered Leaflet GeoJSON layers (translucent fill, soft glow stroke, crisp animated stroke) and returns a LayerGroup so callers can remove later.
geoJsonBounds(geojson) — returns L.LatLngBounds | null for map.fitBounds().

property-boundary.ts — KML/KMZ → GeoJSON. Exports:

interface PropertyBoundary { geojson, kml, fileName, centerLat, centerLng, zoom, areaAcres, bounds }
isBoundaryFile(file) — extension check.
parseBoundaryFile(file) — async; if .kmz, extract doc.kml (or first *.kml) using a small native ZIP reader (parses local file headers + DecompressionStream("deflate-raw") for method-8 entries; handles method-0 stored entries directly). No JSZip dependency.
parseBoundaryKmlString(kml, fileName) — DOMParser, extracts every Placemark > Polygon > outerBoundaryIs > LinearRing > coordinates (and inner rings), builds a single GeoJSON Feature<Polygon> if one polygon found or Feature<MultiPolygon> otherwise.
Computes signed-area-weighted centroid, spherical-excess area in m² (then acres), tight bounds, and suggestZoom(bounds) (residential parcel → 19, large estate → 17, commercial → 15, etc.).
hydrateStoredBoundary({ geojson, centerLat, ... }) — re-builds a PropertyBoundary from persisted customer fields.

Coordinate convention: KML is lng,lat[,alt]; we store GeoJSON in standard [lng, lat]; we use [lat, lng] only for Leaflet APIs.

Notes / gotchas

No JSZip dependency: KMZ extraction uses native DecompressionStream("deflate-raw") (Chrome 80+, Safari 16.4+, Firefox 113+). Falls back to deflate with a synthesized zlib header if deflate-raw isn't present.
GeoJSON namespace is provided transitively by @types/leaflet → @types/geojson, both already in package.json.
No package.json changes required.
Existing KMLParser in client/src/lib/kml-parser.ts is untouched — we add a parallel parser scoped to property boundaries (Polygon-first) so we don't risk regressing the controllers/zones pipeline.
Drizzle decimal columns read back as string; the hydrate helper coerces to number.
Auth: write endpoints reuse requireCustomerEditAccess (admins + billing managers); read endpoint is gated behind requireAuthentication only.
