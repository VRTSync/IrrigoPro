🧾 Master scope paragraph for Replit

We need to redesign the Irrigo monthly invoice detail PDF into a structured, layered billing document system. The PDF should no longer function as a linear report, but instead as a clear invoice justification packet. Page 1 must act as a clean cover page that summarizes the invoice and allows a property manager to quickly validate totals without reading detailed descriptions. All subsequent pages should consist of individual ticket sheets (work orders and billing sheets) that are as close to one-page standalone records as possible, each containing structured work summaries, financial breakdowns, and photos. The final page should provide a concise reconciliation summary that mirrors the invoice totals. The system must eliminate redundant descriptions, enforce strong visual hierarchy, ensure proper pagination, and embed images reliably. The goal is clarity, trust, and ease of validation — not maximum data density.

#PDF1 — Rebuild Cover Page (Invoice Summary Layer)
What & Why

The current Page 1 mixes summary, detail, and narrative content, which forces the user to read instead of confirm. The cover page must become a clean validation dashboard that answers: “Does this invoice make sense?”

Done looks like
Page 1 contains:
logo
company info
customer info
invoice number
billing period
A large, centered Total Invoice Amount
A clean breakdown table (Work Orders / Billing Sheets)
No long descriptions
No ticket-level narrative content
Page is readable in under 10 seconds
Tasks
Remove all descriptions and detailed breakdown content from Page 1
Add a large “TOTAL INVOICE” visual block
Add a clean summary table:
count
labor
parts
total per category
Ensure layout spacing is balanced and uncluttered
Ensure this page always renders as Page 1 and never overflows
Acceptance criteria
A PM can validate totals without scrolling
No duplicate data from later pages appears here
Page can stand alone and still make sense
#PDF2 — Standardize Ticket Page Template (Core Document Layer)
What & Why

Each work order / billing sheet must function as a standalone record. Right now they are close but lack structure and hierarchy.

Done looks like
Each ticket renders as a structured, repeatable “card”
Each ticket page includes:
header (ticket #, invoice #, date, technician, location)
structured “Work Performed” section (bullets, not paragraph)
financial breakdown
parts table
photos
Each ticket is understandable independently
Tasks
Create a reusable TicketPage component for both work orders and billing sheets
Add header block:
Ticket ID
Invoice #
Date
Technician
Location
Convert description into structured bullet list where possible
Add strong “TOTAL” emphasis in financial section
Ensure consistent spacing and layout across all tickets
Acceptance criteria
A single page screenshot of a ticket makes sense without context
No dense paragraph blocks remain
Layout is consistent across all tickets
