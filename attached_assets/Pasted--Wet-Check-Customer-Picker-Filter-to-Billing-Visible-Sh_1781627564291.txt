# Wet-Check Customer Picker ŌĆö Filter to Billing-Visible ("Show in Billing") Customers

**What & Why**
When a field tech starts a new wet check, the customer picker shows **all** customers. It should show only the **active** ones ŌĆö defined as customers whose **"Show in billing"** toggle is on (i.e. `hiddenFromBilling = false`), the Visible/Hidden flag on the customer list.

The picker already *tries* to filter: it requests `/api/customers?active=true`. But the server's `active` filter runs `customers.filter(c => c.isActive !== false)`, and **the customers table has no `isActive` column** ŌĆö so `c.isActive` is `undefined`, `undefined !== false` is always true, and every customer passes. The `active` filter is dead.

The server **does** already support the right filter: `?billingVisible=true` ŌåÆ `customers.filter(c => !c.hiddenFromBilling)`. So the fix is to point the picker at that param. No schema change, no server change.

Single small frontend slice.

---

## Done looks like
- The new-wet-check customer picker lists **only** customers with "Show in billing" = Visible (`hiddenFromBilling = false`).
- Hidden-from-billing customers do **not** appear in the picker.
- No other customer list/screen changes behavior.

## Steps
1. In `artifacts/irrigopro/src/pages/wet-checks/CustomerPickerPage.tsx`, change the customers query from the dead `active` param to the working `billingVisible` param:
   - `queryKey: ["/api/customers", { billingVisible: true }]`
   - `queryFn: () => apiRequest("/api/customers?billingVisible=true")`
   (Currently `["/api/customers", { active: true }]` / `apiRequest("/api/customers?active=true")`.)
2. Leave the rest of the page (status chips, sorting, client-side search) unchanged.
