# BUILD — Realign the Invoice Correction Engine to Source-of-Truth

**Governed by:** PROJECT RULES **Rule 6** (edit at the source ticket; log the activity stream; invoice is a projection) and **Rule 7** (done means proven). Full diagnosis in `AUDIT - Invoice Correction Engine (Source-of-Truth + Activity Stream).md`. **This supersedes** `BUILD - Fix Invoice Reissue (Apply Correction + Exclude Superseded + Versioning)` — that doc's "apply after-values to the invoice, don't touch the ticket" approach is the wrong direction and must not be built.

**What & Why**
The correction flow captures the dispute beautifully but never edits the source ticket and never writes to any ticket's activity stream — so the reissue re-derives from the *unedited* billing sheet and produces the original total (Huntington Trails #04723 reissued at $2,796.09 instead of $2,286.09). Fix the architecture: the correction must **edit the source ticket** (through the canonical recompute), **log that edit to the ticket's activity stream**, and let the invoice re-derive. Server-side + invoices-list UI. Rule 1 multi-tenancy throughout.

## Slice 1 — Reissue applies each correction to its source ticket + logs activity (core)
**File:** `artifacts/api-server/src/routes/invoice-correction-routes.ts` (reissue transaction ~557–681).
Inside the existing reissue `db.transaction`, **before** `computeLiveTotalsFromTickets`, loop the correction lines and apply each to its source ticket (billing sheet / work order / WCB):
- **`zero_line`** → set the disputed subtotal on the ticket (e.g. `laborSubtotal = 0`) and recompute `totalAmount` via the canonical helper (`computeBillingSheetTotal` / `_resyncBillingSheetTotalsTx` / `money()`, `total = parts + labor`) — no hand-rolled sum.
- **`adjust`** → set the ticket to the line's `afterParts` / `afterLabor` and recompute canonically.
- **`exclude`** → unlink the ticket from the invoice (as today).
- **For every applied line, write an activity-stream entry** on that ticket via `recordAuditEvent` (`audit_log`, `target_type` = the ticket type): `action: "billing_correction"`, actor = the correction's approver/creator, `summary` = e.g. "Labor zeroed via invoice correction #04723 — reason: pricing_error, requested by Rob Kaplan", `details` = `{ before, after, correctionId, evidence }`.
- Then the existing `computeLiveTotalsFromTickets` + item-copy produce the corrected total **automatically** (the tickets now hold the edited values). Remove the now-redundant `exclude`-only handling of after-values.
- **Atomicity:** all of this is in the reissue transaction. A correction that is **canceled** (never reissued) must leave every ticket byte-for-byte unchanged — apply happens only at reissue, never during Correct/Document.
- Remove the dead `getOpenCorrectionForInvoice` unlock helper (or wire it — but the reissue-time apply above is the chosen path; don't leave it as dead code per Rule 7).
