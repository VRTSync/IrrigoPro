#116 - Live Billing Exposure Totals
What & Why

The Billing Manager dashboard currently only shows approved (passed-to-billing) totals and has no visibility into tickets still awaiting manager review. This task adds a read-only exposure layer that breaks down financials into Approved, Unapproved, and Total per customer and company-wide, giving the billing manager a complete live picture of the current period without bypassing the approval gate.

Done looks like
The company summary box (top-left) shows three rows: Approved, Unapproved, and Total, replacing the single “Total Unbilled” figure
Each customer card in the left panel shows a three-line breakdown: Approved / Unapproved / Total for the current billing period
Each customer card shows a small status badge: “Up to Date”, “Pending Approval”, “Ready to Bill”, or “Mixed”
Billed tickets (those with an invoiceId) are excluded from all totals
Unapproved tickets cannot be selected, edited, or billed — they are display-only
Totals stay accurate as tickets are created, approved, and billed
Out of scope
Changing the approval workflow
Billing lock logic
Financial calculation logic
Drill-down into individual unapproved tickets
Editing access for unapproved tickets
Changing invoice generation behavior
Tasks

Extend the billing-preview API response — In the /api/customers/billing-preview endpoint, add per-customer approvedTotal, unapprovedTotal, and combinedTotal fields alongside the existing unbilledAmount.

approvedTotal = all work orders and billing sheets in Approved / Passed to Billing state with invoiceId == null
unapprovedTotal = all work orders and billing sheets in Pending Manager Review state with invoiceId == null
combinedTotal = approvedTotal + unapprovedTotal
Exclude all records with invoiceId != null
Apply the same date-range filters already in place. Do not break the existing unbilledAmount field — keep it for backwards compatibility.

Update the company summary box — Replace the single “Total Unbilled” display in the top-left summary card with a three-row breakdown (Approved / Unapproved / Total) computed by summing the new per-customer fields. Keep the count of customers needing billing beneath it, where customers needing billing = customers with approvedTotal > 0.

Update customer cards with three-line breakdown and status badge — On each customer card in the left panel, replace the single “Unbilled Work” badge with a compact three-line breakdown (Approved / Unapproved / Total) and add the status badge logic:

Up to Date = approved 0, unapproved 0
Pending Approval = unapproved > 0, approved = 0
Ready to Bill = approved > 0, unapproved = 0
Mixed = approved > 0, unapproved > 0
Do not increase card height excessively — keep it readable and tight.

Enforce read-only for unapproved tickets — Confirm that no billing selection, edit action, or invoice generation path can be triggered from unapproved (Pending Manager Review) tickets. The new totals are display-only; no new interactive controls should be introduced for unapproved items.
