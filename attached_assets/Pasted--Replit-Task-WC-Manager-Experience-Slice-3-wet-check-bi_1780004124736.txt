## Design Context (locked decisions)

Inherited from WC Manager Experience Slice 0.

**The page is at `artifacts/irrigopro/src/pages/wet-check-billings.tsx`.**
The user-facing path is `/wet-check-billings`. The page is registered for
billing manager, super admin, and company admin (and the irrigation
manager once Slice 5 lands).

**Status badges and `BilledBadge` are the visible delta.** `/billing-sheets`
and `/wet-check-billings` use slightly different badge styling for the
same conceptual states. Slice 1 of this family extracts the
`WetCheckBillingStatusBadge`; this slice goes further and converges with
the BS list page's status pill family.

**Row overflow menus are the action surface on list pages.** `/billing-sheets`
has a `⋯` per-row overflow with Edit / Delete / Reassign etc.
`/wet-check-billings` today has none — opening a row goes straight to the
modal. This slice adds the same overflow pattern so the Slice 5
labor-rate edit and zone-labor edit are reachable as menu items, not
only via deep navigation through the modal.

### Acceptance criteria

- `ListRowOverflowMenu` and `ListPageEmptyState` exist as shared
  components and are consumed by both `/billing-sheets` and
  `/wet-check-billings`.
- Status badges use the shared `WetCheckBillingStatusBadge`
  (extracted by Slice 1) on both pages.
- `/wet-check-billings` has a single filter-chip row replacing the
  two-section collapse.
- Each row on `/wet-check-billings` shows a `⋯` overflow menu with
  View / Edit labor rate / Edit zone labor / Open in QuickBooks,
  role-gated and status-gated correctly.
- Selecting "Edit labor rate" or "Edit zone labor" opens
  `WetCheckBillingViewModal` with the appropriate edit affordance
  pre-focused. (This requires a small `initialAction` prop add to the
  modal — write the change there too.)
- Snapshot/parity tests pass; existing tests continue to pass.
- `pnpm typecheck` and `pnpm test` pass.
