# BUILD — Rebuild Flagged WOs to Approved Estimate + Clean Up Migration List + Remove WO-Corrections Page

**What & Why**
The estimate→WO→completion duplication bug is fixed going forward (completion now replaces items instead of appending). For the handful of *historical* work orders still flagged, we're taking the simple path: **rebuild each to its approved estimate** (bill the quote the customer agreed to) and be done — no per-part review tool needed. This ticket does three things: (A) a new migration that rebuilds the flagged non-billed WOs to their estimate, (B) removes completed/deprecated migrations from the admin list, and (C) deletes the now-unneeded Work Order Correction Review page and its backend. Server-side + minor UI removal; do not touch `customer-billing.tsx`. Multi-tenancy intact; migrations run by Randy from Super Admin.

## Part A — New migration: rebuild flagged WOs to their estimate
**File:** new `artifacts/api-server/src/lib/migrations/repair-wo-match-estimate.ts`, registered in `registry.ts`. Follow the `MigrationDefinition` contract (`check`/`preview`/`run`, per-WO audit log, per-step try/catch that surfaces the real error, acknowledgement gate, idempotent done-marker). **Reuse** the existing detection (`findCandidates`) and `buildRebuiltItemsFromEstimate` + `replaceWorkOrderItemsWithResync` from `repair-wo-items-from-source.ts` — this is essentially that migration with the "must match estimate exactly" guard removed.

- **Target:** every flagged estimate-origin candidate WO that is **not billed** (`invoiceId == null`). For each: delete all `work_order_items`, insert one row per `estimate_item` at the estimate's snapshot price, and recompute header totals (`money()`, `total = parts + labor`) via `replaceWorkOrderItemsWithResync`.
- **Billed WOs (WO-0513-26):** skip with a clear per-WO note ("billed — resolve via invoice reissue"). A migration must not rewrite a billed WO (the resync lock will refuse it anyway).
- **Preview must show, per WO:** `current parts → estimate parts`, the delta, and billed status — so the acknowledgement is an informed decision.
- **Flag WO-1782857855074-481 prominently in the preview:** its excess is **not** duplication ("no duplicate signatures"), so rebuilding it to the estimate **drops ~$1,187 of real, distinct line items** (current $2,968.40 → estimate $1,781.04). Surface this explicitly (e.g., a per-WO warning: "reduction is non-duplicate items — confirm you intend to bill the estimate, not the actuals") so it's a conscious choice, not a silent drop.
- Applies cleanly to WO #26 ($1,607.70), WO-763 ($84.75), WO-48 ($190.42). Idempotent; company-scoped; audit before→after per WO.
