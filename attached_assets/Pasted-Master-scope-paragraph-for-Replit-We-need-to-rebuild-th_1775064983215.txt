Master scope paragraph for Replit

We need to rebuild the monthly invoice detail PDF generator in Irrigo so it is clean, reliable, and easy for customers to match against the invoice. The PDF must clearly separate Work Orders and Billing Sheets, maintain strong visual hierarchy, calculate and display totals correctly, handle page breaks intelligently, embed the correct company logo from a stable source, and render photos reliably without broken links. The final output should read like a professional invoice support packet: page 1 should summarize the invoice and totals, each work record should appear in a clearly delineated section, and the final summary should mirror the invoice for quick verification. This work should prioritize data integrity, predictable layout, and reusable PDF components rather than one-off formatting fixes.

Slice 1 — Audit and map the current PDF generation pipeline
What & why

Before changing layout, identify exactly how the current PDF is built, where totals come from, how logo/image assets are loaded, and why page breaks, photo links, and totals are failing. We need a full map of the PDF data flow and rendering pipeline first.

Replit instructions

Audit the monthly invoice detail PDF generation system end to end. Trace:

where invoice detail data is assembled
where work orders and billing sheets are fetched
how parts/labor/totals are calculated
how the PDF template is structured
how page layout is currently handled
how the logo path is resolved
how photos are loaded into the PDF
whether photos are passed as URLs, file paths, signed URLs, or embedded assets

Identify the exact causes of:

work order totals showing incorrectly
poor section delineation
page breaks splitting content awkwardly
logo failing to load from the correct source
photos showing as broken links or placeholders

Do not redesign yet. Return a written audit and a recommended implementation path.

Done looks like
Replit returns a clear map of the current PDF generation architecture
exact root causes are identified for totals, logo, image rendering, and page break issues
all files/functions involved are listed
recommended next-step changes are identified before code rewrite begins
Acceptance criteria
Can point to the exact total source for both work orders and billing sheets
Can point to the exact asset-loading path for logo and photos
Can explain whether broken images are a data issue, auth issue, URL expiry issue, or PDF rendering limitation
Can explain how page splitting is currently handled and why it fails
