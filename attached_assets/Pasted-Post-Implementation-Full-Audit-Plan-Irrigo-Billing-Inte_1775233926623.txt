Post-Implementation Full Audit Plan — Irrigo Billing Integrity, Approval Flow, and New Data Path
What & Why

All approved slices have now changed the core billing path for work orders and billing sheets. Before this system is trusted in production, I need a full end-to-end audit of the new workflow and all major edge cases.

This is not a partial smoke test. This is a forensic validation pass of the new data path to confirm that the original issues are truly fixed and not still persisting in alternate use cases.

The most important business requirement is:

Once financial and operational data is captured in the field, it must remain intact as it moves up the chain through manager approval, billing manager intake, invoice generation, and billed archival/locking.

This audit must validate:

financial data integrity
approval-gated workflow behavior
correct responsibility handoff
invoice accuracy
QuickBooks payload accuracy
billed lock enforcement
billed visibility in operational views
no hidden recalculation paths still using old logic

The prior audit found that the old system was discarding stored work-order totals and rebuilding them downstream using stripped formulas, and that billed work orders remained editable with weak visibility separation. This audit must specifically prove those old failure modes are gone.

Audit goals

I need the Replit agents to verify all of the following:

Field-submitted data remains financially intact
Customer-specific rates are used correctly
Applied rates are snapshotted correctly at the right stage
Manager approval creates a stable approval record
Billing manager only receives approved tickets
Invoice preview, final invoice, and QuickBooks all use the same authoritative values
Billed tickets become locked
Operational views correctly separate pending review, approved, and billed
No edit path, recalculation path, or hidden endpoint still mutates or rebuilds totals incorrectly
Historical / transitional / edge-case records behave safely
