# Wet Check Convert ‚Äî Auto-Route Completed-in-Field Findings into the Snapshot

**What & Why**
In the wet-check review surface (e.g. WC-81), findings the tech marked **completed in field** can land with **"no billing route"** and a warning to "Open the Manager Workspace findings queue" ‚Äî a queue that has been **removed**. The system already knows these findings belong to the wet check; they should roll into that wet check's WC Snapshot automatically. They don't, because of an **asymmetry between two code paths**:

- **Submit path** (`storage.ts` ~line 7551, `completedInFieldUnrouted`) has a rescue: it catches `techDisposition === 'completed_in_field'` findings **even when `resolution === 'pending'`**, routes them into the wet check's WCB, and stamps `resolution = 'repaired_in_field'`.
- **Convert path** (`storage.ts:8681`, `convertWetCheckToWetCheckBilling`) ‚Äî what **"Approve & Convert"** runs ‚Äî only bills findings where **`resolution !== 'pending'`** (`eligible` filter ~line 8711). It has **no** completed-in-field rescue. So a completed-in-field finding whose resolution was never stamped (auto-bill off at submit, submitted pre-auto-bill, or simply slipped through) is excluded ‚Üí "no billing route."

Result: the manager is told to manually route via a deleted queue, for work that should auto-bill.

**Fix:** give the convert path the same completed-in-field rescue, remove the stale warning, and stop presenting completed-in-field findings as "pending triage."

No schema change.

---

## Slice 1 ‚Äî Convert path: rescue completed-in-field findings (core)

### Files
- `artifacts/api-server/src/storage.ts` ‚Äî `convertWetCheckToWetCheckBilling` (~8681).

### Concrete changes
- In `convertWetCheckToWetCheckBilling`, before/within the `eligible` computation, treat `techDisposition === 'completed_in_field'` findings as repaired-in-field even when `resolution === 'pending'` ‚Äî mirroring the submit path (7551‚Äì7574). Specifically: include findings where `techDisposition === 'completed_in_field' && convertedAt == null && wetCheckBillingId == null && billingSheetId == null && estimateId == null && workOrderId == null && resolution !== 'repaired_in_field'` in the `repaired` set, route them into the wet check's WCB via `_writeRepairedInFieldBilling` (reusing `priorWcbId` so it appends to the one snapshot), and **stamp `resolution = 'repaired_in_field'`** on them so they leave the pending set.
- Keep the existing idempotency/`convertedAt` guards so re-running convert doesn't double-bill.
- Net effect: "Approve & Convert" now auto-routes completed-in-field findings into this wet check's snapshot ‚Äî no manual step.

### Tests
- A wet check with `completed_in_field` findings at `resolution = 'pending'` ‚Üí after convert, each has `wetCheckBillingId` set and `resolution = 'repaired_in_field'`; exactly one WCB for the wet check; totals correct.
- Re-running convert does not create a second WCB or double-bill.
- Findings already routed to estimate/WO/documented are unaffected.

## Slice 3 ‚Äî Stop counting completed-in-field as "pending triage"

### Files
- `artifacts/irrigopro/src/components/wet-check-review/CombinedReviewSurface.tsx` (`unroutedFindings` ~447).
- `artifacts/irrigopro/src/components/manager/wet-check-wizard.tsx` (triage progress / `unroutedCompletedInField`).

### Concrete changes
- Resolve the "3 pending" vs "100% triaged" contradiction: a `completed_in_field` finding requires **no manual triage** ‚Äî it auto-bills on convert. Exclude `techDisposition === 'completed_in_field'` findings from the `unroutedFindings` "needs triage / N pending" count in `CombinedReviewSurface` (they are not awaiting a manager decision). Show them as **"Auto-billed ‚Üí snapshot"** rather than pending.
- Ensure the Step 1 badge and the wizard progress agree: only findings genuinely needing a manager routing decision (no disposition, or `needs_review`) count toward "pending."

### Tests
- A wet check whose only un-billed findings are `completed_in_field` shows **0 pending** in Step 1 and no manual-routing warning; "Approve & Convert" is the clear next action.
- A wet check with a genuinely undecided finding still shows it as pending.
