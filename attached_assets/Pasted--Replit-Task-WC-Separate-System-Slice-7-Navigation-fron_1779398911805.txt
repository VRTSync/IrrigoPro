## Design Context (locked decisions)

These were agreed in conversation and are referenced by every slice in this
rebuild.

**Scope.** Only the wet check ecosystem is affected. Work-order and
standalone BS sheets are unchanged.

**Numbering.** After migration, wet check billings use `WC-YYYY-NNNN`
format. BS-WC- numbering no longer exists.

**Layout for detail view.** Zone-grouped via the existing `WetCheckBillingView`
component, surfaced through the new `WetCheckBillingViewModal` from Slice 4.

**Wet checks feel like their own thing.** Wet check billings are presented
through dedicated nav and pages, not mixed into the billing-sheets surfaces.

**(2) Remove BS-WC prefix-sniffing** in
`artifacts/irrigopro/src/pages/manager-wet-checks.tsx`.

client-side by `billingNumber.startsWith("BS-WC-")` to compute an
"auto-billed today" stat (lines 59–90 approximately). After migration, BS-WC
sheets don't exist anymore — this filter returns 0.

Replace with a fetch from the new endpoint:

```typescript
// Replaces the existing useArrayQuery(["/api/billing-sheets"]) block.
const { data: wetCheckBillings = [] } = useArrayQuery<WetCheckBillingListItem>({
  queryKey: ["/api/wet-check-billings"],
  queryFn: () => fetch('/api/wet-check-billings').then(r => r.json()),
});

// Today's auto-bills: wet check billings with workDate === today
const today = new Date().toDateString();
const autoBilledToday = wetCheckBillings.filter(wcb =>
  new Date(wcb.workDate).toDateString() === today
);
```

Update the JSX that consumed the old filtered list to use `autoBilledToday`
instead. Preserve the same display semantics (count, total amount, link to
detail).

Replace the link at line 142:

```typescript
// OLD:
href="/billing-sheets?prefix=BS-WC-"
// NEW:
href="/wet-check-billings"
```

Drop any imports that become unused after this change (likely `BillingSheet`
type if it was only used here).
