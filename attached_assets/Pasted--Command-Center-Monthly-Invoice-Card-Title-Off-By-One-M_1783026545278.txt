# Command Center ‚Äî Monthly Invoice Card Title Off-By-One Month

**Bug, frontend-only.** In Command Center (`customer-billing.tsx`) ‚Üí a customer's **Monthly Invoices** tab, each invoice card's title shows the **wrong month ‚Äî one month earlier** than the invoice's actual billing month. Example (Wellman Creek): invoice #53707, period 5/31/2026‚Äì6/30/2026, is titled **"May 2026"** on the card. The dedicated **Invoices** page (`invoices.tsx`) shows the same invoice correctly grouped under **"June 2026."** Invoice #76796 (period 4/30‚Äì5/31) is similarly mislabeled "April 2026" on the card but correctly grouped "May 2026" on the Invoices page. Only the card title is wrong ‚Äî invoice #, total, and the Period line on the card are all correct.

**Root cause ‚Äî card title is derived from `periodStart` instead of the invoice's stamped billing month.** `invoice-list.tsx` (the component that renders Command Center's Monthly Invoices cards) computes the title with:

```ts
const formatMonthYear = (date: string) => {
  return new Date(date).toLocaleDateString('en-US', { month: 'long', year: 'numeric' });
};
...
<h3>{formatMonthYear(invoice.periodStart)}</h3>
```

`periodStart` is the *end of the prior* month (e.g. `2026-05-31` for the June cycle), so reading the month off of it gives "May." The Invoices page does **not** do this ‚Äî it groups and labels using the invoice's own stamped `invoiceMonth` / `invoiceYear` fields:

```ts
// invoices.tsx
`${MONTH_NAMES[inv.invoiceMonth - 1]} ${inv.invoiceYear}`
```

`invoiceMonth`/`invoiceYear` are set once at invoice creation (`routes.ts` ~6864-6865, `invoiceMonth: periodStart.getMonth() + 1`) and are already present in the row objects `storage.getInvoices()` returns ‚Äî `GET /api/invoices` (the same endpoint `invoice-list.tsx` already calls) returns them unfiltered via `applyPricingVisibility`. They're just not in `invoice-list.tsx`'s `Invoice` interface or its fetch, so the card falls back to deriving the label from `periodStart` instead.

**Fix:** make the Command Center card title use `invoiceMonth`/`invoiceYear` (the same source of truth `invoices.tsx` and `financial-pulse.tsx` already use), not `periodStart`.

## Done looks like
- A Command Center Monthly Invoices card's title always matches the month/year that the same invoice shows under on the Invoices page (`/invoices`) ‚Äî no divergence for any invoice, past or newly generated.
- The card's Period line (`periodStart`‚Äì`periodEnd`) is unchanged ‚Äî that's already correct.
- No backend changes required; this is a display-only frontend fix.
