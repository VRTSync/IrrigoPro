## Design Context (locked decisions)

These are inherited from the WC Manager Experience Slice 0 audit and the
WC Command Center / WC Billing Integration families.

**The Billing Command Center is not being redesigned.** The customer billing
page (`/billing/command-center` → `customer-billing.tsx`) is the billing
manager's per-customer detail surface. This slice ADDS the WCB tab and the
Total Work line; it does not redesign the page.

**The WCB modal is the action surface.** Clicking a WCB row opens
`WetCheckBillingViewModal`
(`artifacts/irrigopro/src/components/wet-check-billings/wet-check-billing-view-modal.tsx`)
which already has the labor-rate edit pencil for billing-manager-and-above
(WC Billing Integration Slice 5). The new WCB tab's only job is to surface
WCB rows so the modal is reachable.

**Two tab strips exist on this page** — a top strip at `customer-billing.tsx:1129`
and a lower strip at `:1841`. Both must be updated identically. The two
strips serve different layout modes (mobile vs desktop or condensed vs
expanded — verify by reading the surrounding JSX before editing).

### Tests

`customer-billing-wcb-tab.test.tsx`:

```typescript
describe("Customer billing — WCB tab and Total Work line (Slice 1)", () => {
  it("renders 'Wet Check Billings: N' in the Total Work card", () => {
    renderPage({
      wetCheckBillings: [makeWcb({ id: 1 }), makeWcb({ id: 2 })],
    });
    expect(screen.getByText("Wet Check Billings:")).toBeInTheDocument();
    expect(screen.getByText("Wet Check Billings:").nextElementSibling)
      .toHaveTextContent("2");
  });

  it("renders the 'WC Billings (N)' tab on both tab strips", () => {
    renderPage({ wetCheckBillings: [makeWcb({})] });
    const tabs = screen.getAllByRole("tab", { name: /WC Billings/ });
    expect(tabs.length).toBe(2);  // top + bottom strip
    tabs.forEach((t) => expect(t).toHaveTextContent("WC Billings (1)"));
  });

  it("renders WCB rows when the WCB tab is selected", async () => {
    renderPage({
      wetCheckBillings: [
        makeWcb({ id: 1, billingNumber: "WC-2026-1042", totalAmount: "350" }),
      ],
    });
    fireEvent.click(screen.getAllByRole("tab", { name: /WC Billings/ })[0]);
    expect(await screen.findByTestId("wcb-row-1")).toBeInTheDocument();
    expect(screen.getByText("WC-2026-1042")).toBeInTheDocument();
    expect(screen.getByText("$350.00")).toBeInTheDocument();
  });

  it("clicking a WCB row opens the WCB view modal", async () => {
    renderPage({
      wetCheckBillings: [makeWcb({ id: 42 })],
    });
    fireEvent.click(screen.getAllByRole("tab", { name: /WC Billings/ })[0]);
    fireEvent.click(await screen.findByTestId("wcb-row-42"));
    expect(await screen.findByTestId("wcb-view-modal")).toBeInTheDocument();
  });

  it("renders an empty-state card when there are no WCBs", async () => {
    renderPage({ wetCheckBillings: [] });
    fireEvent.click(screen.getAllByRole("tab", { name: /WC Billings/ })[0]);
    expect(await screen.findByText(/No wet check billings yet/)).toBeInTheDocument();
  });

  it("does not change the Unbilled tab count math (already includes WCBs)", () => {
    renderPage({
      unbilledWorkOrders: [{ id: 1 } as any],
      unbilledBillingSheets: [{ id: 2 } as any],
      unbilledWetCheckBillings: [{ id: 3 } as any, { id: 4 } as any],
    });
    expect(screen.getByText(/Unbilled \(4\)/)).toBeInTheDocument();
  });
});
```

### Acceptance criteria

- Both `TabsList`s on `customer-billing.tsx` use `grid-cols-6` and include
  a "WC Billings" / "Wet Check Billings" tab between Billing and Estimates.
- The Total Work card has a "Wet Check Billings: N" row between Billing
  Sheets and Estimates.
- The new tab renders one row per WCB in the customer's billing data,
  using the same card shape as the billing-sheet rows.
- Clicking a row opens `WetCheckBillingViewModal` — the same modal the
  Unbilled tab opens on a WCB row click. Edit pencil (Slice 5) remains
  reachable.
- Empty state renders when `wetCheckBillings.length === 0`.
- All new tests pass. The existing `customer-billing-wet-check.test.tsx`
  continues to pass unchanged.
- No backend changes were made. No new endpoints, no payload shape
  changes.
- `pnpm typecheck` and `pnpm test` pass across the workspace.

### Out of scope

- Adding "Edit" / "Delete" affordances directly on the WCB row (the modal
  carries those).
- Modifying the Unbilled tab's WCB integration (it works per WC Command
  Center Slice 1; not in scope here).
- Financial Pulse widget changes (Slice 2 handles those).
- Renaming the Billing Sheets tab or its content for symmetry with WC
  Billings.
- Adding role gates to the new tab — the customer billing page is
  already role-gated at the route level (billing_manager / company_admin
  / super_admin only).
