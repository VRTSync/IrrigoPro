#117 — Billing Dashboard Revamp & Command Center Separation
What & Why

The current Billing Manager dashboard is doing too much — it mixes execution (selecting and billing tickets) with visibility (understanding where the month stands). This creates clutter and makes it harder to quickly assess financial position.

We are restructuring the billing interface into two distinct layers:

Billing Dashboard (NEW) → Read-only, high-level financial visibility
Billing Command Center (FORMER dashboard) → Execution layer for billing actions

The new Billing Dashboard will act as a financial command overview, answering:

How much is approved and ready to bill?
How much is still pending approval?
Where are the bottlenecks?

The existing dashboard functionality will be moved intact into a new Billing Command Center page, which becomes the only place where billing actions occur.
