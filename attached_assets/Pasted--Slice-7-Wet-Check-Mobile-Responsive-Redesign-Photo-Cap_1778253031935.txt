# Slice 7 — Wet Check Mobile-Responsive Redesign + Photo Capture Fix

**Date:** 2026-05-08  
**Status:** Ready for implementation  
**Files touched:** `src/pages/wet-checks.tsx`, `src/index.css`  
**Risk:** Low — CSS/class changes + two targeted bug fixes, no schema or API changes

---

## Problem Statement

Two issues with the wet check field-tech capture flow:

1. **Layout is not usable on mobile phone screens.** Field techs are using the web app on their phones in the field. The zone grid is 10 columns of tiny squares, buttons are too small to tap reliably, text overflows on narrow screens, and the status chip bar wraps awkwardly.

2. **Photos are not sticking in the system.** Three bugs found in the photo pipeline that cause photos to silently fail or disappear.

## Part B — Photo Capture Fixes (3 Bugs)

### B1. Missing `capture="environment"` on file input

**The bug:** The `<input type="file">` has `accept="image/*"` but no `capture` attribute. On mobile browsers this shows a generic file picker dialog instead of directly launching the rear camera. Techs may be picking old gallery photos or getting confused by the dialog.

**Find** (~line 292):
```jsx
      <input
        ref={inputRef}
        type="file"
        accept="image/*"
        className="hidden"
```

**Replace:**
```jsx
      <input
        ref={inputRef}
        type="file"
        accept="image/*"
        capture="environment"
        className="hidden"
```

`capture="environment"` tells mobile browsers to go straight to the rear-facing camera.

### B2. Missing `wetCheckClientId` on pending-photo PhotoCaptureButton

**The bug:** In the FindingSheet's create-mode (when adding a new finding), the Photo button for attaching photos before save is missing `wetCheckClientId`, `zoneRecordClientId`, and `zoneRecordId`. Without `wetCheckClientId`, the offline photo queue path (`isOfflinePhotosEnabled() && wetCheckClientId`) is NEVER taken. Photos fall through to the legacy direct-upload path, which silently fails on spotty field connections. This is the main reason photos aren't sticking.

**Find** (~line 1795):
```jsx
                    <PhotoCaptureButton
                      wetCheckId={wetCheckId}
                      zoneRecordId={null}
                      findingId={null}
                      skipInvalidate
                      testIdSuffix="pending"
                      onUploaded={(photo) => setPendingPhotos(prev => [...prev, photo])}
                    />
```

**Replace:**
```jsx
                    <PhotoCaptureButton
                      wetCheckId={wetCheckId}
                      wetCheckClientId={wetCheckClientId}
                      zoneRecordId={zoneRecordId}
                      zoneRecordClientId={zoneRecordClientId}
                      findingId={null}
                      findingClientId={null}
                      skipInvalidate
                      testIdSuffix="pending"
                      onUploaded={(photo) => setPendingPhotos(prev => [...prev, photo])}
                    />
```

The same issue affects the pending-photo thumbnails in FindingSheet create mode.
