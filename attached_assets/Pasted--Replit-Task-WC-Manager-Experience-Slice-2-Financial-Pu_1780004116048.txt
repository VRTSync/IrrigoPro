## Design Context (locked decisions)

These are inherited from the WC Manager Experience Slice 0 audit.

**The widget endpoint is `/api/financial-pulse/customer/:customerId/summary`**
— see `financial-pulse-widget.tsx:493`. That endpoint is the single source
of truth for the four customer-detail tiles. The audit MUST start from
the route handler and trace each tile's number back to its query.

**Real-customer spot check.** The user's screenshot shows
"Prospect at Settlers Chase" with $2,075 Billed MTD against $4,853 of
pending work for that customer and $38,651 total unbilled across all
customers. Part A must reconcile the $2,075 figure against the
`invoices` table for that customer in the current calendar month and
report whether the number is correct, off-by-some-known-amount, or
unrelated to the page's other numbers.

**"Billed" is overloaded vocabulary on this page.** The same word means
three different things on the same surface: (1) "Billed MTD" on the
Financial Pulse widget = invoiced+pushed-to-QuickBooks; (2) the
`BilledBadge` on a row = stamped with an `invoiceId`; (3) "0 WC ready"
in the Unbilled Work card uses "ready to bill" implicitly. Whatever
Part B chooses, the fix needs to remove that overload — same word for
three different concepts is the root of the user's confusion.

### Concrete changes

#### Part A — Audit

**(A1) Trace each tile.** Open
`artifacts/irrigopro/src/components/financial-pulse/financial-pulse-widget.tsx`
at line 492 (the `CustomerDetailVariant` function). Read the JSX to
identify the four tiles rendered. Find the matching server handler for
`GET /api/financial-pulse/customer/:customerId/summary` (grep for the
literal path or for `customer/:customerId/summary` across
`artifacts/api-server/src/routes/`). Document each tile in
`docs/financial-pulse-customer-detail-audit.md` using this table:

```
| Tile label  | Source field on response | Query (drizzle / SQL) | Date window function | Customer-scoping predicate | "Counts" definition |
|-------------|--------------------------|-----------------------|----------------------|----------------------------|---------------------|
| Billed MTD  | (e.g.) billedMtd         | from(invoices)...     | gte(invoices.createdAt, startOfMonth) | eq(invoices.customerId, :customerId) | All invoices created this calendar month for this customer, regardless of paid status |
| Billed YTD  | ...                      | ...                   | ...                  | ...                        | ... |
| Money Owed  | ...                      | ...                   | ...                  | ...                        | ... |
| Avg Time to Get Paid | ...             | ...                   | ...                  | ...                        | ... |
```

For each tile, also note: does the query filter out CANCELLED / DRAFT /
VOIDED invoices? Does it use `createdAt`, `txnDate`, or `paidAt` as the
date anchor? Is the customer-scoping predicate at the SQL level or
applied in JS after the query?

**(A3) Write conclusions.** At the bottom of the audit doc, summarize

#### Part B — Fix (post-audit, in a separate PR)

The exact changes depend on Part A's conclusion. Three options:

**Option B-Math-Fix.** If Part A finds a join, window, or scoping bug:
- Fix in the route handler. Mirror the existing handler shape; do not
  introduce a new endpoint.
- Add a regression test in
  `financial-pulse-customer-summary.test.ts` with the spot-check
  customer's data as a fixture: the test fails on `main` and passes
  after the fix.
- No frontend change required.

**Option B-Relabel.** If Part A finds the math is right but the word
"Billed" is overloaded:
- Edit `CustomerDetailVariant` at `financial-pulse-widget.tsx:492+`.
- Rename "Billed MTD" → "Invoiced MTD". Rename "Billed YTD" →
  "Invoiced YTD". Leave "Money Owed" and "Avg Time to Get Paid"
  as-is (those are unambiguous).
- Verify the rename does NOT leak into other variants. The other
  variants live at `financial-pulse-widget.tsx:~242, ~376, ~409` and
  use the same `label="Billed MTD"` literal. They are NOT in scope —
  if you find the label is shared via a constant, **do not change the
  constant**; pass a variant-specific label prop at the
  `CustomerDetailVariant` call site instead.

**Option B-Tooltip.** Add hover/focus tooltips to each tile explaining
the metric in plain language:
- "Billed MTD: invoices created and pushed to QuickBooks this month.
  Does not include queued unbilled work."
- "Money Owed: total of unpaid invoices for this customer."
- Use the existing `Tooltip` / `TooltipProvider` / `TooltipContent`
  primitives (already imported elsewhere in this file or available from
  `@/components/ui/tooltip`).

**Combinations are expected.** A common Part A finding will be "math is
right but the word is overloaded AND first-timers need plain-language
explanations." That's Option B-Relabel + B-Tooltip together.
