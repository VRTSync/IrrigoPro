# Replit Task — WC Manager Experience, Slice 7 (Unified wet-check list page: one tool, role-gated views)

This is a single self-contained task. The **Design Context** below is locked
decisions from `WC Manager Experience - Slice 0 Unification Audit and Plan.md`
— it is reference material so you understand the intent. The actual work for
this task is in the **Slice 7** section at the bottom. Do not deviate from any
file paths, function names, schema field names, or payload shapes specified.
If you believe a deviation is necessary, surface it and stop rather than
proceeding.

> **This is the centerpiece of the cross-role unification.** Today
> `manager-wet-checks.tsx` (171 LOC), `admin-wet-checks.tsx` (588 LOC),
> and `wet-checks.tsx` (18 LOC, a barrel re-export) are three separate
> files built in isolation. The user's direction is "one tool, different
> views based on role." After this slice, there is one component, three
> role-gated lenses, and the legacy files are gone.

> **Pattern: shared elements must not vary by role.** Columns, badges,
> row layout, filter bar shape, empty state copy — all identical
> regardless of role. Only the COLUMNS rendered and the ACTIONS available
> are role-gated. If a stylistic difference creeps in, the unification
> fails.

> **Largest LOC change in the family.** Plan for a careful migration with
> the legacy pages deleted at the end, not in the middle.

#### Step 1 — Audit (PR description artifact, no production code)

Read all three legacy pages. Capture the deltas in a table in the PR
description:

```
| Element / Feature       | manager-wet-checks | admin-wet-checks | wet-checks (picker) | Resolution in unified |
|-------------------------|--------------------|------------------|---------------------|------------------------|
| Data fetch              | /api/wet-checks?status=submitted | /api/wet-checks (all) | /api/customers | Use /api/wet-checks with role-aware default filter |
| Columns                 | WC#, customer, age, $ | WC#, customer, company, tech, status, findings | (n/a — customer cards) | Unified set with role-gated company column |
| Filters                 | (none)             | company, status, tech | (none) | Single filter bar: status / customer / tech (admin gets company) |
| Bulk select             | no                 | yes (delete)     | no                  | Role-gated: company_admin + super_admin only |
| Empty state             | "No reviews"       | "No wet checks"  | customer picker     | One unified empty state with customer-picker affordance |
| Row click target        | /manager/wet-checks/:id | /wet-checks/:id | /wet-checks/c/:customerId/new | All → /wet-checks/:id (canonical) |
| KPI tiles               | yes (HeaderStrip)  | no               | no                  | Optional — keep only if user explicitly requests (likely defer) |
```

The Resolution column drives the rest of the slice.

#### Step 2 — `WetChecksListPage` component

```typescript
// artifacts/irrigopro/src/pages/wet-checks/WetChecksListPage.tsx

interface WetChecksListPageProps {
  // No props — role and userId come from getCurrentUser().
}

export default function WetChecksListPage(): JSX.Element {
  const user = getCurrentUser();
  const role = user?.role ?? "field_tech";

  // Role-aware defaults
  const defaultStatusFilter = useMemo(() => {
    if (role === "irrigation_manager") return "submitted,pending_manager_review";
    if (role === "billing_manager") return "approved_passed_to_billing,billed";
    return ""; // company_admin / super_admin see all
  }, [role]);

  const [statusFilter, setStatusFilter] = useState(defaultStatusFilter);
  const [customerFilter, setCustomerFilter] = useState<number | null>(null);
  const [techFilter, setTechFilter] = useState<number | null>(null);
  const [companyFilter, setCompanyFilter] = useState<number | null>(null); // super_admin only

  const wetChecks = useArrayQuery<WetCheck>({
    queryKey: ["/api/wet-checks", { status: statusFilter, customerId: customerFilter, technicianId: techFilter, companyId: companyFilter }],
    // ...
  });

  if (wetChecks.length === 0 && !statusFilter && !customerFilter && !techFilter) {
    // No wet checks at all — show the customer picker affordance.
    return <WetChecksEmptyStateWithPicker />;
  }

  return (
    <PageContainer>
      <PageHeader
        title="Wet checks"
        subtitle={subtitleForRole(role, wetChecks.length)}
        actions={role !== "billing_manager" ? <NewWetCheckButton /> : undefined}
      />
      <PageContent>
        <DismissibleHelp guideId="wc-list-first-time" variant="info">
          Click any row to open the review page. Use the filter chips to
          narrow by status, customer, or technician.
        </DismissibleHelp>

        <WetCheckFilterBar
          status={statusFilter}
          onStatusChange={setStatusFilter}
          customer={customerFilter}
          onCustomerChange={setCustomerFilter}
          tech={techFilter}
          onTechChange={setTechFilter}
          company={role === "super_admin" ? companyFilter : null}
          onCompanyChange={role === "super_admin" ? setCompanyFilter : undefined}
        />

        <table className="w-full">
          <thead>
            <tr>
              <th>WC #</th>
              <th>Customer</th>
              <th>Tech</th>
              <th>Date</th>
              <th>Status</th>
              <th>Findings</th>
              {role === "super_admin" ? <th>Company</th> : null}
              <th aria-label="Actions" />
            </tr>
          </thead>
          <tbody>
            {wetChecks.map((wc) => (
              <WetCheckRow
                key={wc.id}
                wc={wc}
                role={role}
              />
            ))}
          </tbody>
        </table>

        {(role === "company_admin" || role === "super_admin") ? (
          <BulkActionBar wetChecks={wetChecks} />
        ) : null}
      </PageContent>
    </PageContainer>
  );
}
```

#### Step 3 — Shared `WetCheckRow` component

```typescript
// artifacts/irrigopro/src/components/wet-checks/wet-check-row.tsx
export interface WetCheckRowProps {
  wc: WetCheck;
  role: UserRole;
}

export function WetCheckRow({ wc, role }: WetCheckRowProps) {
  const navigate = useNavigate();
  const onClick = () => navigate(`/wet-checks/${wc.id}`);
  const actions = useRoleGatedActions(wc, role);

  return (
    <tr
      onClick={onClick}
      className="cursor-pointer hover:bg-gray-50"
      data-testid={`wc-row-${wc.id}`}
    >
      <td><span className="font-mono text-xs">WC-{wc.id}</span></td>
      <td className="truncate">{wc.customerName}</td>
      <td className="text-gray-600">{wc.technicianName ?? "—"}</td>
      <td className="text-gray-600">{formatDate(wc.workDate)}</td>
      <td><WetCheckStatusBadge status={wc.status} /></td>
      <td className="text-gray-600">{wc.findingCounts?.total ?? 0}</td>
      {role === "super_admin" ? <td>{wc.companyName ?? "—"}</td> : null}
      <td onClick={(e) => e.stopPropagation()}>
        <ListRowOverflowMenu actions={actions} testIdSuffix={wc.id} />
      </td>
    </tr>
  );
}
```

`useRoleGatedActions` is a small helper that returns the action list
appropriate for the role:

```typescript
function useRoleGatedActions(wc: WetCheck, role: UserRole): ListRowAction[] {
  const navigate = useNavigate();
  return [
    { label: "Open review", icon: <Eye className="w-4 h-4" />, onSelect: () => navigate(`/wet-checks/${wc.id}`) },
    { label: "View PDF", icon: <FileText className="w-4 h-4" />, onSelect: () => window.open(`/api/wet-checks/${wc.id}/pdf`), hidden: !wc.pdfReady },
    { label: "Reassign technician", icon: <UserPlus className="w-4 h-4" />, onSelect: () => openReassignDialog(wc), hidden: !["company_admin", "super_admin"].includes(role) },
    { label: "Delete", icon: <Trash className="w-4 h-4" />, destructive: true, onSelect: () => openDeleteDialog(wc), hidden: !["company_admin", "super_admin"].includes(role) },
  ];
}
```

#### Step 4 — Shared `WetCheckFilterBar`

```typescript
// artifacts/irrigopro/src/components/wet-checks/wet-check-filter-bar.tsx
export function WetCheckFilterBar({
  status, onStatusChange,
  customer, onCustomerChange,
  tech, onTechChange,
  company, onCompanyChange,
}: WetCheckFilterBarProps) {
  return (
    <div className="flex gap-2 mb-3 flex-wrap">
      <FilterChipGroup>
        <FilterChip active={!status} onClick={() => onStatusChange("")}>All</FilterChip>
        <FilterChip active={status.includes("submitted")} onClick={() => onStatusChange("submitted")}>Submitted</FilterChip>
        <FilterChip active={status.includes("pending_manager_review")} onClick={() => onStatusChange("pending_manager_review")}>Pending review</FilterChip>
        <FilterChip active={status.includes("approved_passed_to_billing")} onClick={() => onStatusChange("approved_passed_to_billing")}>Approved</FilterChip>
        <FilterChip active={status.includes("billed")} onClick={() => onStatusChange("billed")}>Billed</FilterChip>
      </FilterChipGroup>
      {customer != null || onCustomerChange ? <CustomerPicker value={customer} onChange={onCustomerChange} /> : null}
      {tech != null || onTechChange ? <TechPicker value={tech} onChange={onTechChange} /> : null}
      {onCompanyChange ? <CompanyPicker value={company} onChange={onCompanyChange} /> : null}
    </div>
  );
}
```

  check list surfaces.
- Three legacy files (`manager-wet-checks.tsx`,
  `admin-wet-checks.tsx`, and the old wet-checks customer picker) are
  deleted or absorbed.
- App.tsx routes are consolidated; legacy paths 301-redirect.
- Shared row component, filter bar, and status badge are used by all
  three role lenses.
- Role-gated columns and actions work as specified.
- DismissibleHelp from Slice 6 is consumed for first-timer guides.
- Snapshot parity test passes (row chrome identical across roles).
- Feels-fast bounds mirror Slice 6's: < 800ms first interactive paint
  on a primed cache; no skeleton flash on second visits.
- All new and updated tests pass.
- `pnpm typecheck` and `pnpm test` pass.
