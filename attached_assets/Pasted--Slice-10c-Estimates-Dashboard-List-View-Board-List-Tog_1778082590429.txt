## Executive Summary

Slice 10b shipped the board as the only view on `/estimates`. 10c adds the alternative ŌĆö a polished, sortable list view with color-coded status badges that match the board's column tints ŌĆö plus a segmented Board/List toggle in the page header that persists across reloads. The two views share data, filters, and modal entry points; switching is instant and stateless except for the toggle preference.

10c also closes the wizard loop introduced by 10a's draft state. The estimate creation wizard gets a second CTA on Step 3 ŌĆö "Save as draft" ŌĆö that creates an estimate with `internalStatus: 'draft'`, lands it in the Drafts column without going through the billing manager review queue, and lets the irrigation manager come back to finish or submit it later. Editing a draft from the board opens the same wizard with a dual-CTA footer: "Save changes" keeps it as a draft; "Submit for Approval" transitions it to `pending_approval` via the new transition endpoint.

**Slice 10c ships when:** the page renders both views and the toggle works on desktop and mobile; the toggle preference persists across reloads; both views show the same filtered data; the list view's column headers sort correctly; the wizard's "Save as draft" button creates a draft estimate that appears in the Drafts column; editing a draft surfaces the dual-CTA footer; submitting a draft via the dual-CTA correctly transitions it to Pending review; the Slice 7 queue and customer approval flow continue to work.

## Why Now

Three motivations.

First, **a list view is a strict requirement for users who manage many estimates at once.** A kanban is excellent for at-a-glance lifecycle reads but inefficient for sorting by amount, scanning a single customer's full history, or comparing dollar values across statuses. The list view fills that gap. Color-coordinated badges keep the lifecycle-at-a-glance benefit when scanning the table.

Second, **drafts are the wizard's missing exit.** Today the only way to close the wizard with unsaved work is to discard. Field operators commonly start an estimate at a job site and finish it later from a desk; without a Save-as-draft path, they re-enter the data twice. 10a made `'draft'` a real state; 10c is what makes that state reachable from the UI.

Third, **the dual-CTA footer in edit-mode-on-a-draft is the bridge between drafts and the rest of the lifecycle.** Without it, a draft is a one-way street (you'd have to do a second edit pass after submitting). The dual-CTA gives the irrigation manager the choice on each save.

## What `/estimates` Becomes

After Slice 10c, `/estimates` has a Board/List toggle in the header. Both views render the same filtered estimate set.

**Page header strip** (replaces the 10b strip):
- Title "Estimates" + subtitle "{N} active ┬Ę last 90 days".
- Right side, in order: Board/List toggle (segmented control), Filter button, `+ New Estimate` button.

**Board view** (10b):
- Unchanged from Slice 10b. Filter, columns, expired strip, column-expand all work the same.

**List view** (new in 10c):
- A single sortable table replaces the column grid. Columns:
  | Column | Behaviour |
  |---|---|
  | Customer | Customer name + project name in muted secondary type below. Click sort: alpha. |
  | Amount | `$X,XXX`. Right-aligned. Click sort: numeric desc/asc. |
  | Status | Color-coded badge using same column tints as board. Click sort: lifecycle order (Drafts ŌåÆ Pending review ŌåÆ Sent ŌåÆ Approved ŌåÆ Rejected ŌåÆ Expired). |
  | Age | `Nd` / `Nw`. Click sort: chronological. Default sort. |
  | Actions | Three-dot menu ŌĆö "Open", "Edit", and (for expired only) "Resend" placeholder (10d wires it). |
- Row hover: `bg-gray-50`. Click anywhere on a row except the actions menu ŌåÆ opens the detail modal (same as board cards).
- Default sort: Date desc. Sort indicator is a small chevron next to the active column header.
- Empty state: centered placeholder with "+ New Estimate" link.

**Both views share**:
- The same `useQuery` for `/api/estimates`.
- The same Filter popover (Customer multi-select + Status chips, identical from 10b).
- The same `EstimateDetailModal` open-flow.
- The same expired callout ŌĆö but the rendering shape differs:
  - Board: horizontal callout strip below columns (10b).
  - List: rows with `lifecycleStatus = 'expired'` appear inline in the table with their badge styled with a strikethrough text-decoration as in the visual mock.

**Wizard changes**:
- **Step 3 footer adds a "Save as draft" secondary button** alongside "Submit for Approval". Layout: Back (left), Save as draft (right-secondary), Submit for Approval (right-primary).
- "Save as draft" calls the existing `POST /api/estimates` with `internalStatus: 'draft'` in the payload. Toast "Saved to drafts." Wizard closes.
- **Edit mode footer**: when the loaded estimate has `lifecycleStatus === 'draft'`, the footer shows two primary-styled CTAs: "Save changes" (left) and "Submit for Approval" (right). Save changes does a `PUT /api/estimates/:id` with the current edits, status unchanged. Submit for Approval does the same `PUT` plus a follow-up `POST /api/estimates/:id/transition { action: 'submit_for_review' }`. Toast on the latter: "Submitted for review."
- For estimates not in `draft` status, edit-mode footer is unchanged from Slice 8 (single primary "Save changes").

**Toggle behavior**:
- Stored in `localStorage` under key `estimates_view_preference` (string: `'board'` | `'list'`). Default: `'board'`.
- Read at page mount via `safeGet('estimates_view_preference', 'board')`.
- Written on every toggle click via `safeSet('estimates_view_preference', value)`.
- The toggle is a segmented control: two pill-shaped buttons inside a rounded container; selected button gets `bg-gray-100`, unselected is plain.

## Risks and Rollback

- **Sort state lost on view switch.** Switching from Board to List loses any in-progress List sort if the list state is unmounted. Mitigation: keep the list's sort state at the page level (or in `EstimateList`) so view switching doesn't discard it. Alternative: don't unmount, just `display: none` ŌĆö but that hurts initial render.
- **Wizard's dual-CTA footer ambiguous on edit-mode-of-draft.** A user may not realize "Submit for Approval" is the action that progresses the estimate. Mitigation: add helper text below the buttons in edit-mode-of-draft: "Save changes keeps this as a draft. Submit for Approval sends it to the billing manager." Small `text-xs text-gray-500`.
- **localStorage key collision.** If `estimates_view_preference` clashes with another key, the existing `safeGet`/`safeSet` defensive parsing should still keep things from blowing up. Verify by listing existing keys (or just choose a clearly namespaced key).
- **Status sort by lifecycle order is unintuitive.** Users may expect alphabetical. Mitigation: keep lifecycle order as the default since it tells a flow story (drafts ŌåÆ ... ŌåÆ rejected); a future polish slice can add an alpha-vs-lifecycle toggle in the column header if anyone asks.
- **Edit-from-draft regression risk.** The dual-CTA path performs PUT + transition in sequence. If the transition fails after the PUT succeeds, the user sees a "Saved" toast and then a "Submit failed" toast ŌĆö confusing. Mitigation: do the transition first if the user clicked Submit for Approval; only do the PUT after the transition validates the row. Or do them as a single composite request ŌĆö but the simplest is: do PUT first, wait, then call transition; on transition failure, show a toast that says "Saved as draft, but couldn't submit for review. Try again." The state is recoverable.
- **Mobile list view at <380px viewport.** Stacked cards work, but if the customer name is long, the row gets tall. Mitigation: truncate customer name to 30 chars on mobile cards.

**Rollback:** `git revert` the slice commit. Existing 10b board view still works; the toggle and list view simply disappear; the wizard's Save-as-draft button disappears with the wizard reverting to single-CTA. Drafts created before the revert remain in the database with `internal_status = 'draft'` ŌĆö they'll show in the board's Drafts column from 10b but can no longer be created from the UI until 10c lands again. Acceptable rollback state.
