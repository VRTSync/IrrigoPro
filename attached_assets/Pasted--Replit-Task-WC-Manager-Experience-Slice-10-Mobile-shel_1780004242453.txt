## Design Context (locked decisions)

Inherited from WC Manager Experience Slice 0 (Slice 10 section) and
the audit's role-split answer (locked: shell-based, not role-based).

**The mobile app is at `artifacts/irrigopro-mobile/`.** The home tab
is `app/(tabs)/index.tsx`. The tab layout is at `app/(tabs)/_layout.tsx`
with three tabs: Today, Wet Checks, Profile.

**The "Today" tab today is field-tech-shaped.** It renders a list of
the day's work orders pulled from `/api/work-orders?assigned=me`.
That's correct for field techs. For an irrigation manager logging in
on mobile, that view answers the wrong question — the manager isn't
looking for "my assigned WOs," they're looking for "what action do I
need to take today?"

**The irrigation manager's mobile-relevant actions are:**
- Start a new wet check on a customer (the field-but-also-managers
  responsibility).
- Create a work order on the fly.
- Assign a tech to an open WO.
- Quick-review a recently submitted wet check (lighter than the web
  review page; mostly "approve or kick back to look at later").

**Shared role, distinct shells.** The role is `irrigation_manager` on
both web and mobile. The mobile shell looks at the role and renders
the manager dashboard; the web shell does the same. The decision is
made in the SHELL, not in a separate API or a different role.

function FieldActionButton({ icon, label, onPress }: { icon: keyof typeof Feather.glyphMap; label: string; onPress: () => void }) {
  const colors = useColors();
  return (
    <Pressable onPress={onPress} style={[styles.fab, { backgroundColor: colors.card, borderColor: colors.border }]}>
      <Feather name={icon} size={20} color={colors.primary} />

function ActionTile({ label, count, subtitle, onPress }: { label: string; count: number; subtitle: string | null; onPress: () => void }) {
  const colors = useColors();
  if (count === 0) {
    return (
      <View style={[styles.tile, { backgroundColor: colors.card, borderColor: colors.border, opacity: 0.6 }]}>
        <Text style={[styles.tileLabel, { color: colors.mutedForeground }]}>{label}</Text>
        <Text style={[styles.tileCount, { color: colors.mutedForeground }]}>0</Text>
      </View>
    );
  }
  return (
    <Pressable onPress={onPress} style={[styles.tile, { backgroundColor: colors.card, borderColor: colors.border }]}>
      <View style={{ flex: 1 }}>
        <Text style={[styles.tileLabel, { color: colors.foreground }]}>{label}</Text>
        {subtitle ? <Text style={[styles.tileSubtitle, { color: colors.mutedForeground }]}>{subtitle}</Text> : null}
      </View>
      <Text style={[styles.tileCount, { color: colors.primary }]}>{count}</Text>
      <Feather name="chevron-right" size={20} color={colors.mutedForeground} />

}

function subtitleForAge(oldestAgeHours: number | null | undefined): string | null {
  if (oldestAgeHours == null) return null;
  if (oldestAgeHours < 1) return "Just now";
  if (oldestAgeHours < 24) return `Oldest: ${Math.floor(oldestAgeHours)}h ago`;
  return `Oldest: ${Math.floor(oldestAgeHours / 24)}d ago`;
}

**(3) `oldestAgeHours` requires a small server-side addition.** The
existing `/api/manager-workspace/status-strip` from Slice 8 returns
counts only. Extend it to also return `oldestAgeHours` per indicator:

```typescript
// In manager-workspace-routes.ts, augment the status-strip response:
const oldestAgeHours = (rows: any[], statusSet: Set<string>): number | null => {
  const matching = rows.filter((r) => statusSet.has(r.status));
  if (matching.length === 0) return null;
  const oldest = matching.reduce((min, r) => {
    const t = new Date(r.createdAt).getTime();
    return t < min ? t : min;
  }, Infinity);
  return (Date.now() - oldest) / 3_600_000;
};

res.json({
  indicators: {
    wcsPendingReview: {
      count: wcsPendingReviewCount,
      oldestAgeHours: oldestAgeHours(wetChecks, ACTIVE_WC),
    },
    // ... and so on for each indicator
  },
});
```

This is a small additive backend change to Slice 8's endpoint. If

The web `/manager-workspace` page's `StatusTile` ignores the

### Acceptance criteria

- Mobile Today tab renders ManagerTodayScreen for irrigation_manager,
  company_admin, super_admin.
- Mobile Today tab continues to render FieldTechTodayScreen for
  field_tech (no behavior regression).
- ManagerTodayScreen displays four field-action buttons and three
  action tiles (Wet checks pending review / WOs awaiting approval /
  Findings needing routing).
- Action tile counts come from `/api/manager-workspace/status-strip`.
- Zero-count tiles render in a muted style.
- `oldestAgeHours` is computed server-side and surfaced as a "Oldest:
  Xh ago" subtitle on the tiles.
- All tests pass.
- Mobile linting (`pnpm --filter irrigopro-mobile lint`) passes.

### Out of scope

- New mobile screens for routes that don't yet exist (note them in
  the PR; spec separately).
- Touching the Wet Checks tab or Profile tab.
- Changing the tab bar structure (Today / Wet Checks / Profile stays).
- Push notifications.
- Offline-first behavior for the manager dashboard
  (the queries refetch when online; offline behavior is the same
  pattern as elsewhere in the mobile app).

### Dependencies

- Slice 8 (`/api/manager-workspace/status-strip` endpoint) — hard
