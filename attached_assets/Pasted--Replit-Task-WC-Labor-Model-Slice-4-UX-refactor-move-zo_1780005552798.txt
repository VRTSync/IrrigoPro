# Replit Task — WC Labor Model, Slice 4 (UX refactor — move zone labor edit into Zone Details)

This is a single self-contained task. The **Design Context** below is locked
decisions from `WC Labor Model - Slice 0 Per-Finding Labor Audit and Plan.md`.
The actual work for this task is in the **Slice 4** section at the bottom.
Do not deviate from any file paths, function names, schema field names, or
payload shapes specified. If you believe a deviation is necessary, surface
it and stop rather than proceeding.

> **Frontend-only slice. No schema, no backend changes.** The endpoint
> stays at `PATCH /api/wet-check-billings/:id/zone-labor`
> (`routes.ts:9653`). The model stays per-zone authoritative. Only the
> edit affordance moves from the misplaced top panel to inline near
> each zone's subtotal.

> **Pattern: editor next to data.** The "Zone Repair Labor" panel today
> is a list at the top of the modal, far from the zones it describes.
> The user reports it "means nothing to them" because it requires
> mental cross-referencing. Inline editing next to the zone subtotal
> in the Zone Details section keeps the labor edit visually adjacent
> to the parts that compose it.

#### (2) Build the inline editor

```typescript
// artifacts/irrigopro/src/components/wet-check-billings/zone-labor-edit-inline.tsx
//
// Inline per-zone repair labor editor. Lives in each zone's block in
// the WCB view's Zone Details section, immediately above the
// ZONE SUBTOTAL row.

import { useState } from "react";
import { useMutation, useQueryClient } from "@tanstack/react-query";
import { apiRequest } from "@/lib/queryClient";
import { Pencil, RotateCcw } from "lucide-react";
import { Input } from "@/components/ui/input";
import { Button } from "@/components/ui/button";
import { Badge } from "@/components/ui/badge";

interface ZoneLaborEditInlineProps {
  wcbId: number;
  zoneRecordId: number;
  valueHours: string;
  manuallySet: boolean;
  laborRate: number;
  canEdit: boolean;
}

export function ZoneLaborEditInline({
  wcbId,
  zoneRecordId,
  valueHours,
  manuallySet,
  laborRate,
  canEdit,
}: ZoneLaborEditInlineProps) {
  const [editing, setEditing] = useState(false);
  const [local, setLocal] = useState(valueHours);
  const queryClient = useQueryClient();

  const saveMut = useMutation({
    mutationFn: (hours: string) =>
      apiRequest(`/api/wet-check-billings/${wcbId}/zone-labor`, "PATCH", {
        zoneRecordId,
        repairLaborHours: hours,
      }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["/api/wet-check-billings", wcbId] });
      setEditing(false);
    },
  });

  const resetMut = useMutation({
    // If the backend doesn't have a reset-to-auto endpoint yet, add one
    // as part of this slice — small addition, same role gate. Pattern
    // from storage.ts:7737 (resetZoneRepairLabor — field-tech tier).
    // The billing-tier version is a parallel method that skips
    // assertWetCheckEditableByTech and only checks
    // wcb.invoiceId IS NULL.
    mutationFn: () =>
      apiRequest(`/api/wet-check-billings/${wcbId}/zone-labor/reset`, "POST", { zoneRecordId }),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["/api/wet-check-billings", wcbId] });
    },
  });

  const hoursNum = parseFloat(valueHours || "0");
  const formattedHours = `${hoursNum.toFixed(2)} hr${hoursNum === 1 ? "" : "s"}`;
  const dollarValue = (hoursNum * laborRate).toFixed(2);

  if (!canEdit) {
    return (
      <div
        className="grid grid-cols-12 text-sm py-2 border-t border-gray-100"
        data-testid={`zone-labor-readonly-${zoneRecordId}`}
      >
        <div className="col-span-7 flex items-center gap-2 text-gray-700">
          <span>Zone Labor</span>
          {!manuallySet ? <Badge variant="outline" className="text-xs">auto</Badge> : null}
        </div>
        <div className="col-span-3 text-right text-gray-700">{formattedHours}</div>
        <div className="col-span-2 text-right text-gray-700">${dollarValue}</div>
      </div>
    );
  }

  return (
    <div
      className="grid grid-cols-12 text-sm py-2 border-t border-gray-100 items-center"
      data-testid={`zone-labor-row-${zoneRecordId}`}
    >
      <div className="col-span-7 flex items-center gap-2 text-gray-700">
        <span>Zone Labor</span>
        <Badge variant="outline" className="text-xs">
          {manuallySet ? "manual" : "auto"}
        </Badge>
        {manuallySet ? (
          <button
            onClick={() => resetMut.mutate()}
            disabled={resetMut.isPending}
            className="text-xs text-gray-500 hover:text-gray-700 inline-flex items-center gap-1"
            data-testid={`zone-labor-reset-${zoneRecordId}`}
          >
            <RotateCcw className="w-3 h-3" /> Reset to auto
          </button>
        ) : null}
      </div>
      <div className="col-span-3 text-right">
        {editing ? (
          <form
            onSubmit={(e) => { e.preventDefault(); saveMut.mutate(local); }}
            className="inline-flex items-center gap-1 justify-end"
            data-testid={`zone-labor-form-${zoneRecordId}`}
          >
            <Input
              type="number"
              step="0.25"
              min="0"
              value={local}
              onChange={(e) => setLocal(e.target.value)}
              className="w-20 h-7 text-right"
              autoFocus
              data-testid={`zone-labor-input-${zoneRecordId}`}
            />
            <Button type="submit" size="sm" disabled={saveMut.isPending}>Save</Button>
            <Button type="button" size="sm" variant="ghost" onClick={() => { setEditing(false); setLocal(valueHours); }}>
              Cancel
            </Button>
          </form>
        ) : (
          <button
            onClick={() => { setLocal(valueHours); setEditing(true); }}
            className="text-gray-700 hover:underline underline-offset-2"
            data-testid={`zone-labor-edit-${zoneRecordId}`}
          >
            {formattedHours}
            <Pencil className="inline w-3 h-3 ml-1 text-gray-400" />
          </button>
        )}
      </div>
      <div className="col-span-2 text-right text-gray-700">${dollarValue}</div>
    </div>
  );
}
```

If `POST /api/wet-check-billings/:id/zone-labor/reset` doesn't
already exist, add it as a small backend additive change in this

```typescript
// routes.ts — alongside the existing /zone-labor PATCH
app.post("/api/wet-check-billings/:id/zone-labor/reset", requireAuthentication, async (req, res) => {
  const cid = requireCompanyId(req, res); if (!cid) return;
  const role = req.authenticatedUserRole;
  if (!["billing_manager", "company_admin", "super_admin"].includes(role)) {
    res.status(403).json({ message: "Forbidden" }); return;
  }
  const wcbId = parseInt(req.params.id, 10);
  const zoneRecordId = parseInt(req.body?.zoneRecordId, 10);
  if (!Number.isFinite(wcbId) || !Number.isFinite(zoneRecordId)) {
    res.status(400).json({ message: "Invalid ids" }); return;
  }
  try {
    const result = await storage.resetWcbZoneRepairLabor(wcbId, zoneRecordId, cid);
    if (!result) { res.status(404).json({ message: "Not found" }); return; }
    res.json(result);
  } catch (e: any) {
    // classify + log following the existing pattern
    res.status(500).json({ message: "Couldn't reset zone labor" });
  }
});
```

### Acceptance criteria

- The "Zone Repair Labor" panel and the `EditZoneRepairLabor`
  component are deleted.
- A `ZoneLaborEditInline` row appears in each zone block in Zone
  Details, immediately above the ZONE SUBTOTAL.
- The row shows the hours, the dollar value, and an auto/manual
  badge. The reset-to-auto link appears only when `manuallySet`.
- The Repairs Summary cell is display-only (no edit affordance).
- The role gate matches the existing endpoint
  (billing_manager+); field tech and read-only roles see the
  display only.
- Locked WCBs (`billed` / has `invoiceId`) show display only,
  regardless of role.
- The `POST /api/wet-check-billings/:id/zone-labor/reset` endpoint
  exists and recomputes WCB totals on success.
- All new tests pass; existing modal tests pass.
- `pnpm typecheck` and `pnpm test` pass.
