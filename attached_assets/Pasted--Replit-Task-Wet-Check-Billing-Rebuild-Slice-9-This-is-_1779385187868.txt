## Design Context (locked decisions)

These were agreed in conversation and are referenced by every slice in this
rebuild. Do not deviate from any of them.

**Scope.** Only billing sheets whose `billingNumber` starts with `BS-WC-` are
affected. Work-order and other billings keep the existing flat-parts layout.

**Source of truth for the new layout.** `wetCheckFindings` joined to
`wetCheckZoneRecords`. Each finding already has `billingSheetId`,
`zoneRecordId`, `issueType`, `partId`, `partName`, `partPrice`, `quantity`,
`laborHours`. No data migration is needed — historical BS-WC sheets render
correctly from existing data.

**Layout.** Zone-grouped. Each zone displays as: zone header
(`<controllerLetter> <zoneNumber>` + issue display label), part lines, per-zone
repair labor row showing `hours × rate = total`, per-zone subtotal. A single
wet-check-level "Inspection Labor" line appears once per sheet.

**Repair labor entry.** Per-zone, stepper UI, 0.25-hour increments, 0.25-hour
minimum. Free-form decimal inputs are not permitted in the new flow.

**Labor rate.** Always the per-customer global rate (`customers.laborRate`),
snapshotted onto the billing sheet at creation (`billing_sheets.appliedLaborRate`).
Never reprice based on later customer-rate changes. Existing snapshot logic is
preserved.

**Issue type display.** Always use `issueTypeConfigs.displayLabel` for the
customer's company. Never display the raw `issueType` field. If no config row
exists for a `(companyId, issueType)` pair, fall back to title-cased
`issueType.replace(/_/g, " ")`.

**$0.00 line item suppression on PDF.** Suppress when `lineTotal === 0 AND
laborHours === 0 AND NOT isLaborOnly`. Labor-only findings (qty=0, partPrice=0,
noPartNeeded=true) are kept because they represent real work performed.

**PDF treatment.** Repairs Summary table at the top (zone label → subtotal).
Each zone wrapped in a div with `page-break-inside: avoid`. Labor math shown on
PDF (`hours × rate = total`). Labor subtotal split into "Inspection Labor" +
"Repair Labor". Issue type display labels everywhere.

> **Note for Slice 9 specifically:** The "Inspection Labor + Repair Labor" split
> language above describes the *original* design. A later in-cycle change
> (Task #766) deliberately collapsed that split into a single "Irrigation Labor"
> line. Slice 9 must respect Task #766 — do **not** re-introduce the split.

**Issue type config normalization.** `issueTypeConfigs.defaultLaborHours` must
be a multiple of 0.25 with a 0.25 minimum. The seed and admin validation must
enforce this.

**Safety patch.** `deleteWetCheck` must refuse when any of the wet check's
findings has `billingSheetId IS NOT NULL`, even if the BS is not yet on an
invoice. Today's blocker only checks invoice linkage, which leaves a back door
that the new renderer would silently break against.

**No PDF backward compatibility.** All wet check billings live in the current
billing cycle (the wet check feature didn't exist before this cycle). There is
no "old vs new" PDF reconciliation to handle. Regenerating an existing BS-WC PDF
in the new format is the correct behavior.
