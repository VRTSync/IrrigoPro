# Wet Check Page — Fix Unclickable Tabs + Remove Stale Nav Links

Two small frontend fixes on the new Wet Check System page. No backend, no schema.

---

## Fix 1 — Tabs are not clickable (wouter query-string quirk)

**Root cause (`artifacts/irrigopro/src/pages/wet-checks/WetCheckSystemPage.tsx`, ~line 443):**
```js
const [location, navigate] = useLocation();
const params = new URLSearchParams(location.split("?")[1] ?? "");
const tab = parseTab(params.get("tab"));
```
wouter's `useLocation()` returns the **pathname only** — it does not include the query string. So `location.split("?")[1]` is always `undefined`, `tab` is always the default (`"all"`), and clicking a tab calls `setTab` → `navigate("/wet-checks?tab=…")` which updates the URL but the component **never re-reads** the tab. Result: tabs appear unclickable (the content never switches).

### Done looks like
- Clicking All / Needs Review / Approved switches the visible tab.
- Deep-linking / refreshing on `?tab=needs-review` (and `approved`) lands on the right tab.
- The active tab styling reflects the current tab.

### Steps
1. Read the query string **reactively** instead of from `location`. Use wouter's `useSearch` hook:
   ```js
   import { useLocation, useSearch } from "wouter";
   // ...
   const [, navigate] = useLocation();
   const search = useSearch();                       // reactive query string
   const tab = parseTab(new URLSearchParams(search).get("tab"));
   function setTab(t: Tab) {
     navigate(`/wet-checks?tab=${t}`, { replace: true });
   }
   ```
   (Alternatively, hold the active tab in `useState`, initialized from the URL once, and update both state and URL in `setTab` — either approach fixes it; `useSearch` keeps the URL as the source of truth.)
2. Verify the three `TabButton`s pass `isActive={tab === "…"}` and `onClick={() => setTab("…")}` against the now-correct `tab` value.

### Tests
- Clicking each tab renders that tab's content and marks it active.
- Loading `/wet-checks?tab=approved` directly opens the Approved tab.
